Мониторинг серверов с помощью New Relic

May 23, 2016 12:40 · 150 words · 1 minute read monitoring new relic

New Relic — пример еще одной отличной системы мониторинга, с которой обязательно стоит познакомиться. Давайте разберемся как настроить базовый мониторинг серверов с помощью New Relic!

Инструкции по установке и настройке необходимого ПО есть на сайте, и они не должны вызывать никаких проблем. Например, для ОС Debian/Ubuntu это будет выглядеть так:

echo deb http://apt.newrelic.com/debian/ newrelic non-free | sudo tee /etc/apt/sources.list.d/newrelic.list

wget -O- https://download.newrelic.com/548C16BF.gpg | sudo apt-key add -

Обновляем информацию о доступных пакетах и устанавливаем нужный:

sudo apt-get update

sudo apt-get install newrelic-sysmond

Добавляем свой лицензионный ключ в конфиг:

sudo nrsysmond-config --set license_key=1234567890abcdefghijklmnopqrstuvwxyz

Примечание. Конечно же здесь необходимо ввести свой лицензионный код.

Запускаем процесс мониторинга сервера:

sudo service newrelic-sysmond start

В конфигурационном файле /etc/newrelic/nrsysmond.cfg также (по желанию) можно изменить параметр hostname.

Спустя некоторое время данные о состоянии сервера будут доступны в аккаунте New Relic.
